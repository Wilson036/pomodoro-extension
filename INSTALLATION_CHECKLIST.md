# ✅ 安裝檢查清單

使用此清單確保番茄鐘 Chrome 插件正確安裝和運行。

## 📋 安裝前準備

### 環境檢查
- [ ] Node.js 已安裝 (v16+)
  ```bash
  node --version
  ```
  
- [ ] npm 已安裝
  ```bash
  npm --version
  ```

- [ ] 使用 Chrome 瀏覽器（版本 88+）

## 🚀 安裝流程

### ✅ 步驟 1：準備專案
- [ ] 已下載或 clone 專案
- [ ] 在終端機中進入專案目錄
  ```bash
  cd pomodoro-extension
  ```
- [ ] 確認目錄結構正確
  ```bash
  ls -la
  # 應該看到：package.json, src/, public/ 等
  ```

### ✅ 步驟 2：安裝依賴
- [ ] 執行安裝命令
  ```bash
  npm install
  ```
- [ ] 等待安裝完成（約 30-60 秒）
- [ ] 確認沒有紅色錯誤訊息
- [ ] 確認 `node_modules` 資料夾已建立
  ```bash
  ls -la node_modules
  ```

### ✅ 步驟 3：建構插件
- [ ] 執行建構命令
  ```bash
  npm run build
  ```
- [ ] 等待建構完成（約 10-20 秒）
- [ ] 確認 `dist` 資料夾已建立
  ```bash
  ls -la dist
  ```
- [ ] 確認 `dist` 資料夾包含以下文件：
  - [ ] manifest.json
  - [ ] popup.html
  - [ ] popup.js
  - [ ] background.js

### ✅ 步驟 4：載入到 Chrome
- [ ] 打開 Chrome 瀏覽器
- [ ] 在網址列輸入：`chrome://extensions/`
- [ ] 開啟右上角的「開發人員模式」開關
  - 確認開關變成藍色
- [ ] 點擊「載入未封裝項目」按鈕
- [ ] 瀏覽並選擇專案中的 `dist/` 資料夾
- [ ] 確認插件出現在列表中
- [ ] 確認插件已啟用（開關為藍色）
- [ ] 檢查是否有錯誤訊息（紅色文字）
- [ ] 確認工具列出現插件圖標

## ⚠️ 問題排查

### 如果建構失敗：
1. - [ ] 檢查 Node.js 版本
   ```bash
   node --version  # 應該 >= v16
   ```
2. - [ ] 刪除並重新安裝依賴
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   npm run build
   ```
3. - [ ] 查看錯誤訊息並 Google 搜尋解決方案

### 如果插件無法載入：
1. - [ ] 確認「開發人員模式」已開啟
2. - [ ] 確認選擇的是 `dist` 資料夾（不是專案根目錄）
3. - [ ] 查看 Chrome 擴展頁面顯示的具體錯誤
4. - [ ] 確認 `dist` 資料夾內有 manifest.json
5. - [ ] 檢查 manifest.json 的 JSON 格式是否正確

### 如果通知不顯示：
1. - [ ] Chrome 設定 → 隱私權和安全性 → 網站設定 → 通知
2. - [ ] 確認 Chrome 通知已啟用
3. - [ ] 系統設定 → 通知
4. - [ ] 確認系統通知已啟用
5. - [ ] 確認沒有「請勿打擾」模式
6. - [ ] 重新載入插件並再次測試

### 如果音效無法播放：
1. - [ ] 檢查系統音量
2. - [ ] 檢查 Chrome 音訊設定
3. - [ ] 確認沒有靜音
4. - [ ] 先與網頁互動（點擊按鈕）再測試音效
5. - [ ] 在 Chrome 設定中檢查音訊權限
